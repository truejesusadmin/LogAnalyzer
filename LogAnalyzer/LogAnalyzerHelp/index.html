Introduction

LogAnalyzer application provides a way to analyze log files. Or to be more exact, the application displays any plain-text file. When loading a file, LogAnalyzer checks if the lines start with a timestamp in a common format, such as 2015-03-22 10:11:23.456 etc. In that case it takes the input file as a log and groups the lines according to their timestamp value.

The application numbers the lines during loading. Each line number then becomes a unique identifier for a line, and it doesn’t change during any other actions such as filtering, copying or grouping.


Opening a log

There are several ways to open the log files:

  • In Finder, define LogAnalyzer as the default application for opening log file type. After that, all you have to do is to click to open the file in Finder.
  • Run LogAnalyzer and press ⌘+O.
  • Drag and drop a file to the LogAnalyzer window.
  • Drag and drop a file to the LogAnalyzer icon in dock if the icon is present.
  • Select text in other application (such as XCode) and drag and drop it at the LogAnalyzer application window.

If you wish to append a file to an existing log shown in the application window, simply drag it to the LogAnalyzer application window. Lines from the new file will be appended to the end of the open file, ie. they will not be inserted based on their timestamp.


Analyzing a log

The basic way to analyze a log in LogAnalyzer is filtering. There are two ways to filter, which can be combined:

  • Searching – input the text in the search bar.
  • Make a selection of lines.

Selected lines will be highlighted and available for additional work. But first let’s describe both methods to select filtered lines in more detail:

Searching by entering text in the search bar

Text in the search bar can be entered by either typing it in (the search starts automatically), or by pasting text using standard keyboard shortcut (⌘+V).

If you want to search a text shown in the log, you can display a window with the details of a line:

  • Double click a selected line.
  • Hit ⏎ key on a selected line.

A popup window will appear with the details of the line. You can highlight the searched text in the window using mouse, then:

  • After hitting ⏎, the popup windows closes and the highlighted text is transferred to the search box and search action starts automatically.
  • By hitting Escape the popup window closes without starting a search action.

Highlighting a from-to selection

It is possible to click the line number column. A popup window with line numbers and icons “from row” and “to row” will appear. Alternatively it is possible to use keyboard shortcuts.

  • ⌥+⌘+▽ for choosing the first line of the selection.
  • ⌥+⌘+△ for choosing the last line of the selection.

Cancelling a from-to selection

You can cancel a selection by selecting a row below the “to” line or above the “from” line.


Working with selection in the log

  • To switch between highlighted lines and all lines, hit button .
  • To invert the selection, hit button .
  • To delete highlighted lines, hit button .
  • Hit ⌘+C for copying highlighted lines, ⌘+N for opening a new window and ⌘+V for pasting the selected lines to the new window.
  • By dragging highlighted or normal rows away from the LogAnalyzer window, a new window will be opened and the dragged lines will be copied into this new window.
  • The above-mentioned ways to move lines can be used repeatedly. Newly added lines will be inserted between the existing ones to a position determined by their row number.

Moving between the log lines

  • The arrow keys ▽ and △ can be used to move to the next or previous line in the log.
  • The keys ⌘+▽ and ⌘+△ can be used to move to the next or previous highlighted line in the log.
  • To delete highlighted lines, hit ⌘+Del or button .
  • Hit ⌘+C for copying highlighted lines, ⌘+N for opening a new window and ⌘+V for pasting the selected lines to the new window.
  • By dragging highlighted or normal rows away from the LogAnalyzer window, a new window will be opened and the dragged lines will be copied into this new window.
  • The above-mentioned ways to move lines can be used repeatedly. Newly added lines will be inserted between the existing ones to a position determined by their row number.

Sorting log

By hitting ⌘+⌥+S you can sort the log file by the number of matching characters in log lines. Lines with the largest number of identical characters are on the top of newly opened window.


Using favorites

While you are browsing log, you cat put selected log item to favorites list by hitting ⌘+⏎ or I key. By hitting ⌘+I, or by pushing the button, you can show or hide the favourites list. By selecting a log item in the favouritest list, the log view will be synchronized.


Remote logging

The LogAnalyzer application supports remote logging. It uses the Bonjour protocol to receive continuous stream of log messages from a client that cannot for some reason be connected to the computer with XCode through a cable.


RemoteLogger framework

To enable remote logging on the iOS client side, it is necessary to implement the RemoteLogger logging framework. The implementation into iOS applications requires the following steps:

  • Add RemoteLogger.framework to the project. Also, add it to “Embeded Binaries”.
  • At an appropriate place, such as the application delegate, configure the logging type.

A sample iOS project that demonstrates using the RemoteLogger framework is available at:

https://github.com/vnevyhosteny/RLoggerDemo

How to establish remote logging

In order for remote logging to work, it is necessary that the iOS device has Wi-Fi turned on and is on the same network as the receiving computer running LogAnalyzer. Alternatively, the iOS device needs Bluetooth turned on, enabled Bluetooth hotspot and the Mac with LogAnalyzer has to be connected to the iOS Bluetooth hotspot.

Then all that is needed in LogAnalyzer is hitting ⌘+B, or the button .

LogAnalyzer starts to search for clients, and if there is an iOS device with an application having the RemoteLogger framework implementation, the iOS device will show a connection request. After the request is accepted, the logs will be shown in LogAnalyzer (with minor delay).
